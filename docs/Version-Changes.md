# Изменения версий

Этот документ отделяет два разных уровня изменений:

- **опубликованный релиз `1.4.0`** на NuGet;
- **изменения текущего кода репозитория** относительно `1.4`, зафиксированные в ветке `1.5 Preview`.

## Executive summary

Если смотреть глазами конечного пользователя:

- `1.4.0` — подтверждённый опубликованный пакет на NuGet, доступный для установки и распространяемый под MIT; `[confirmed by package metadata]`
- текущий код репозитория ушёл дальше `1.4.0` и уже содержит матрицы, пароль, новые перегрузки `CreateCell`, дополнительный API чтения из памяти и новый компрессионный слой `DESMini`; `[confirmed by git history]`
- при этом опубликованный пакет и текущий `csproj` репозитория расходятся по части платформенных метаданных, поэтому документацию нужно читать с учётом контекста источника. `[confirmed by package metadata] [confirmed by code]`

## Что изменилось для конечного пользователя в текущем коде после `1.4`

### Главное

- появились матрицы `int[,]` как новый поддерживаемый тип данных; `[confirmed by code]`
- появилась парольная проверка при чтении; `[confirmed by code]`
- добавлены типизированные перегрузки `CreateCell(...)`; `[confirmed by code]`
- добавлен in-memory итератор `NextData(...)`; `[confirmed by code]`
- добавлен `NumberInfoInt()`; `[confirmed by code]`
- формат стал использовать короткие токены типов и новые служебные маркеры; `[confirmed by code]`
- появился новый матричный компрессор `DESMini`. `[confirmed by code]`

## Изменения публичного API

### Сравнение `1.4` и текущего кода

| Элемент API | Было в `1.4` | Стало в текущем коде | Impact |
| --- | --- | --- | --- |
| `Password(string)` | нет | да | Можно ограничивать чтение паролем |
| `CreateCell(int)` | нет | да | Проще и безопаснее добавлять `int` |
| `CreateCell(double)` | нет | да | Проще добавлять `double` |
| `CreateCell(float)` | нет | да | Можно передавать `float` напрямую |
| `CreateCell(string)` | нет | да | Проще добавлять строки |
| `CreateCell(bool)` | нет | да | Проще добавлять bool |
| `CreateCell(int[,])` | нет | да | Новый сценарий: матрицы |
| `MatrixData(string)` | нет | да | Можно получить матрицу по маркеру |
| `NextData(int = -1)` | нет | да | Появилось чтение из памяти после загрузки |
| `NumberInfoInt()` | нет | да | Не нужно парсить строку количества |

### Что осталось прежним

- центральный класс `DEObject`;
- конструктор `DEObject(...)` с теми же базовыми параметрами;
- методы `Save()`, `OpenAll()`, `OpenNext()`, `ResetData()`, `Association()`, `CloneDataDE()`, `RecreateCell(...)`; `[confirmed by code]`
- однофайловый и двухфайловый режимы как концепция. `[confirmed by code]`

## Изменения формата файла

| Изменение | Воздействие на пользователя | Action required |
| --- | --- | --- |
| Полные токены типов заменены на короткие (`integer` -> `i`, и т.д.) | Файлы новой ветки компактнее | Для межверсинной совместимости используйте новый reader |
| Добавлен маркер `"p"` | Появилась парольная проверка | Перед чтением защищённого файла вызывайте `Password(...)` |
| Добавлен тип `"m"` | Можно сохранять матрицы | Используйте `CreateCell(int[,])` и `MatrixData(...)` |
| Добавлен маркер `"l"` в one-file | Сжимаются серии `int` | Ничего не делать, но учитывать несовместимость со старым reader |
| Новый разделитель `"/"` вместо старого длинного маркера | Упрощение однофайлового формата | Новый reader совместим с обоими вариантами |

## Изменения алгоритма сжатия

В `1.4` отдельного модуля `DESMini` не было. В текущем коде:

- добавлен класс `DESMini`;
- добавлен trie-компрессор для словарей байтовых последовательностей;
- добавлена nibble-индексация;
- добавлены `int-delta` и `byte-delta`;
- введён выбор между single-key и sequence mode. `[confirmed by code]`

Практический эффект:

- библиотека теперь имеет специализированный компрессор под матрицы;
- однофайловый режим косвенно выигрывает и на длинных сериях `int`, потому что они упаковываются через матричный блок.

## Breaking changes

### Подтверждённые риски

- старый код `1.4` не знает короткие токены `"i"`, `"d"`, `"s"`, `"b"`, `"f"`, `"m"`; `[confirmed by code]`
- старый код `1.4` не знает маркер `"p"` и матричный тип `"m"`; `[confirmed by code]`
- старый код `1.4` не знает маркер `"l"` однофайлового режима; `[confirmed by code]`
- следовательно, файлы, записанные новым кодом, не следует считать обратно совместимыми со старым reader `1.4`. `[confirmed by code]`

### Что осталось обратно совместимым

Новый reader умеет читать:

- старые строковые имена типов;
- старый разделитель однофайлового режима. `[confirmed by code]`

То есть совместимость направлена в сторону "новый код читает старый формат", а не наоборот.

## Migration notes

### Если вы уже используете `1.4.0`

- оставьте текущий writer, если вам критична совместимость со старыми рантаймами;
- переходите на новую ветку только если вам нужны матрицы, пароль или обновлённый API;
- для миграции старых файлов сначала откройте их новым кодом, затем пересохраните.

### Если вам нужен безопасный путь обновления

1. Возьмите тестовый набор данных `1.4`.
2. Откройте его новым кодом.
3. Сразу выполните `Save()` в новом формате.
4. Повторно откройте пересохранённые файлы новым экземпляром `DEObject`.
5. Сравните значения и, если используете матрицы, проверьте `MatrixData(...)`.

## Release notes draft

Ниже черновик релизных заметок для текущей ветки после `1.4`, основанный только на коде и diff.

### Draft: GDELib 1.5 Preview

- Добавлена поддержка матриц `int[,]` в публичном API `DEObject`.
- Добавлены типизированные перегрузки `CreateCell(...)` для `int`, `double`, `float`, `string` и `bool`.
- Добавлен метод `MatrixData(...)` для извлечения матриц после чтения.
- Добавлен метод `NextData(...)` для последовательного чтения уже загруженных значений из памяти.
- Добавлен метод `NumberInfoInt()`.
- Добавлена парольная проверка файлов через SHA-256-хэш.
- Обновлён формат хранения: короткие токены типов, новые служебные маркеры `p`, `m`, `l`.
- Реализован новый модуль `DESMini` для сжатия матриц и серий целых чисел.
