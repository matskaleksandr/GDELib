# Изменения в версии 1.4.0

Этот документ посвящён именно релизной версии `GDELib 1.4.0`.

## Executive summary

`GDELib 1.4.0` — это опубликованная версия библиотеки, доступная через NuGet и ориентированная на практическое использование в приложениях .NET. Основная модель работы остаётся простой и понятной:

- библиотека предоставляет один основной класс `DEObject`;
- данные сохраняются в формате SVE;
- поддерживаются однофайловый и двухфайловый режимы;
- библиотека подходит для прикладного хранения данных и файлов.

## Что важно для пользователя версии 1.4.0

В релизе `1.4.0` библиотека представлена как готовый пакет для подключения в проект и для распространения через NuGet.

### Поддержка платформ

Версия `1.4.0` доступна для:

- `.NET 6.0`
- `.NET Standard 2.1`
- `.NET Framework 4.7.2`

Это делает релиз удобным не только для современных проектов, но и для приложений, которым нужна совместимость с более широким набором целевых платформ.

## Сравнение с опубликованной версией 0.1.4

По внешнему подходу библиотека сохраняет тот же базовый сценарий работы:

- центральный класс `DEObject`;
- добавление данных через `CreateCell(...)`;
- сохранение через `Save()`;
- чтение через `OpenAll()` и `OpenNext(...)`.

Главное практическое отличие релиза `1.4.0` — более полноценная публикация пакета и более широкая платформенная доступность.

## Что изменилось для конечного пользователя

| Изменение | Что это даёт |
| --- | --- |
| Публикация `1.4.0` на NuGet | Удобная установка в проект через пакетный менеджер |
| Поддержка нескольких целевых платформ | Можно использовать библиотеку в большем числе .NET-проектов |
| Сохранение единой модели работы через `DEObject` | Документация и примеры остаются простыми и понятными |

## Публичный API

Для прикладного использования основа API версии `1.4.0` выглядит так же:

- `DEObject(...)`
- `CreateCell(...)`
- `Save()`
- `OpenAll()`
- `OpenNext(...)`
- `ResetData(...)`
- `Dell(...)`
- `RecreateCell(...)`
- `NumberInfo()`
- `Association(...)`
- `CloneDataDE(...)`

Это хороший сценарий для разработчика: библиотека остаётся компактной и не требует изучения большого набора классов.

## Формат хранения и режимы работы

Для пользователя в `1.4.0` особенно важны два режима:

- двухфайловый — структура и данные раздельно;
- однофайловый — один основной контейнер.

С точки зрения практического применения это значит, что библиотеку можно подстроить либо под более явную файловую структуру, либо под формат одного контейнера.

## Breaking changes

На уровне пользовательского сценария работы релиз `1.4.0` сохраняет знакомую модель использования библиотеки через `DEObject`.

При обновлении пакета всё равно полезно перепроверить:

- установку через NuGet;
- режим хранения, который используется в проекте;
- сценарии чтения файловых ячеек.

## Migration notes

Если проект уже использует более ранний пакет, переход на `1.4.0` обычно выглядит так:

1. обновить пакет до `1.4.0`;
2. проверить примеры записи и чтения;
3. убедиться, что рабочий режим хранения выбран правильно;
4. перепроверить файловые ячейки, если в проекте сохраняются файлы.

## Release notes draft

Ниже — компактный вариант заметок к релизу `1.4.0`.

### Draft

- GDELib 1.4.0 доступна через NuGet.
- Библиотека распространяется как open-source проект под лицензией MIT.
- Библиотека предназначена для удобного сохранения и чтения данных в формате SVE.
- Внешний API построен вокруг класса `DEObject`.
- Поддерживаются однофайловый и двухфайловый режимы.
- Версия опубликована для `.NET 6.0`, `.NET Standard 2.1` и `.NET Framework 4.7.2`.
- Установка: `Install-Package GDELib -Version 1.4.0`.
