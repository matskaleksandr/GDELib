# API `DEObject`

`DEObject` — центральный класс библиотеки GDELib и фактически вся её пользовательская API-поверхность. Через него создаётся контейнер данных, в него добавляются ячейки, а затем тот же класс отвечает за сохранение, открытие и последовательное чтение.

## Почему работа начинается именно с `DEObject`

Простыми словами:

- `DEObject` заменяет пользователю и "документ", и "сериализатор", и "ридер";
- у библиотеки нет отдельного публичного builder/read-only reader слоя;
- поэтому жизненный цикл работы почти всегда строится вокруг одного объекта.

Технически:

- класс объявлен как `public class DEObject`; `[confirmed by code]`
- внутреннее состояние хранится в списке `YList`;
- выбор режима записи делается через поле `TOne`, а фактическая запись/чтение делегируется внутреннему `DESaver`. `[confirmed by code]`

## Типовой жизненный цикл

```csharp
using GDELib;

var de = new DEObject(path);
de.CreateCell(123);
de.CreateCell("value");
de.Save();

var reopened = new DEObject(path);
string[] values = reopened.OpenAll();
```

Полезно различать два режима чтения:

- `OpenAll()` / `OpenNext()` — читают данные с диска;
- `NextData()` — итерируется по уже загруженному в память набору. `[confirmed by code]`

<a id="конструктор-deobject"></a>
## Конструктор `DEObject`

```csharp
public DEObject(
    string path_,
    bool _TOne = false,
    string _NameData = "data.sve",
    string _NameStruct = "struct.sve",
    string _pathcash = "pathcash")
```

### Что это

Конструктор создаёт рабочий контейнер и настраивает, где именно библиотека будет хранить данные и в каком режиме она будет работать.

### Когда использовать

- при первой записи набора данных;
- при повторном открытии уже сохранённых файлов;
- при переключении между однофайловым и двухфайловым режимом через новый экземпляр.

### Параметры

| Параметр | Смысл | Комментарий |
| --- | --- | --- |
| `path_` | Папка хранения | Именно здесь библиотека ищет и пишет `data.sve`/`struct.sve` |
| `_TOne` | Режим хранения | `false` — двухфайловый, `true` — однофайловый |
| `_NameData` | Имя файла данных | По умолчанию `data.sve` |
| `_NameStruct` | Имя файла структуры | По умолчанию `struct.sve`; в однофайловом режиме не используется при сохранении |
| `_pathcash` | Путь к папке кэша | Если оставить `"pathcash"`, используется `Environment.CurrentDirectory` |

### Что делает конструктор дополнительно

- создаёт внутренний `DESaver`;
- при необходимости пытается создать папку `cashfile` внутри кэш-каталога. `[confirmed by code]`

### Ограничения и ошибки

- если у процесса нет прав на создание папки кэша, ошибка выводится в `Console`, а не оформляется как отдельное исключение библиотеки; `[confirmed by code]`
- конструктор не проверяет заранее существование или корректность самих файлов данных.

### Лучший практический способ использования

- всегда передавайте явный путь хранения;
- для продакшена лучше также явно передавать `_pathcash`, а не полагаться на текущую директорию процесса.

<a id="passwordstring-passw"></a>
## `Password(string passw)`

```csharp
public void Password(string passw)
```

### Что это

Метод задаёт пароль, который затем участвует в сохранении и чтении файла.

### Зачем нужен

Если пароль задан до `Save()`, библиотека записывает SHA-256-хэш пароля в поток данных и затем сверяет его при `OpenAll()`/`OpenNext()`. `[confirmed by code]`

### Как вызвать

```csharp
var de = new DEObject(path);
de.Password("my-secret");
de.CreateCell("protected");
de.Save();
```

### Что вернёт

Ничего. Метод только меняет состояние объекта в памяти.

### Ограничения

- данные не шифруются; хранится только хэш пароля; `[confirmed by code]`
- если вы забудете вызвать `Password(...)` перед чтением защищённого файла, чтение завершится `null` и сообщением в консоль;
- пароль действует только в рамках конкретного экземпляра `DEObject`.

### Практика

Ставьте пароль до записи и до чтения. Рассматривайте его как проверку доступа, а не как криптографическую защиту.

<a id="семейство-createcell"></a>
## Семейство `CreateCell`

### Что это

`CreateCell` добавляет очередную ячейку в конец текущего набора.

### Поддерживаемые перегрузки

```csharp
public void CreateCell(string types, object data)
public void CreateCell(int data)
public void CreateCell(double data)
public void CreateCell(float data)
public void CreateCell(string data)
public void CreateCell(bool data)
public void CreateCell(int[,] matrix)
```

### Простой смысл

Каждый вызов создаёт ещё один элемент в последовательности. Порядок вызовов — это будущий порядок чтения.

### Какую перегрузку выбирать

| Перегрузка | Когда использовать | Особенность |
| --- | --- | --- |
| `CreateCell(int)` | Обычное целое число | Самый прямой и безопасный вариант |
| `CreateCell(double)` | Значение с плавающей точкой | Хранится как `double` |
| `CreateCell(float)` | Есть исходный `float` | Внутри переводится в тип ячейки `double` |
| `CreateCell(string)` | Текстовые данные | Пустая строка допускается |
| `CreateCell(bool)` | Флаги и переключатели | При сохранении пишется строковое представление `True/False` |
| `CreateCell(int[,])` | Матрица целых чисел | Актуально для текущего кода репозитория |
| `CreateCell(string, object)` | Когда тип приходит динамически | Поддерживает строковые типы `int`, `double`, `string`, `bool`, `file` |

### Примеры

```csharp
de.CreateCell(10);
de.CreateCell(12.75);
de.CreateCell("text");
de.CreateCell(true);
de.CreateCell(new int[,] { { 1, 2 }, { 3, 4 } });
de.CreateCell("file", @"C:\Temp\report.txt");
```

### Что вернёт

Ничего. Новая ячейка добавляется во внутренний список.

### Ограничения и подводные камни

- универсальная перегрузка `CreateCell(string, object)` не поддерживает строковый тип для матриц; для матриц есть отдельная перегрузка; `[confirmed by code]`
- если передать неподдерживаемое строковое имя типа, метод просто не добавит ячейку и не бросит исключение; `[confirmed by code]`
- `float` не имеет отдельного формата хранения, он попадает в тип `double`. `[confirmed by code]`

### Практика

Предпочитайте типизированные перегрузки. Строковая версия нужна в основном для обвязок, UI-конфигураторов и generic-кода.

<a id="matrixdatastring-code"></a>
## `MatrixData(string code)`

```csharp
public int[,] MatrixData(string code)
```

### Что это

Возвращает матрицу по строковому маркеру, который был получен при чтении.

### Когда использовать

Когда `OpenAll()`, `OpenNext()` или `NextData()` вернули значение вида `matrix_N`.

### Как вызвать

```csharp
string[] values = de.OpenAll();
int[,] matrix = de.MatrixData(values[0]);
```

### Что вернёт

Матрицу `int[,]`, уже загруженную во внутренний список.

### Ограничения и ошибки

- метод ждёт строку формата `matrix_<index>`; проверка формата минимальная; `[confirmed by code]`
- если индекс неверный, можно получить стандартное исключение доступа к массиву/списку. `[inferred from implementation]`

### Практика

Не собирайте строку вручную, а используйте именно тот маркер, который вернуло чтение.

<a id="save"></a>
## `Save()`

```csharp
public void Save()
```

### Что это

Записывает текущий набор ячеек на диск.

### Когда использовать

- после заполнения контейнера;
- после изменения данных через `CreateCell`, `ResetData`, `RecreateCell`, `Dell`;
- после переключения режима через `Association(...)`, если нужен новый формат хранения.

### Как работает

- если `TOne == false`, вызывается двухфайловый `Save()` внутреннего `DESaver`;
- если `TOne == true`, вызывается однофайловый `SaveOne()`. `[confirmed by code]`

### Что вернёт

Ничего.

### Ограничения и ошибки

- ошибки внутри сериализации в основном перехватываются и печатаются в `Console`; `[confirmed by code]`
- в однофайловом режиме есть внутренняя оптимизация серий `int`, которая зависит от порядка ячеек; `[confirmed by code]`
- сохранение файловых ячеек использует zip через кэш-папку.

### Практика

После `Save()` создайте новый экземпляр `DEObject` и проверьте результат через `OpenAll()`. Это лучший способ быстро поймать несовместимость пути, пароля или режима хранения.

<a id="openall"></a>
## `OpenAll()`

```csharp
public string[] OpenAll()
```

### Что это

Полностью читает содержимое файла и одновременно заполняет текущий `DEObject` данными в памяти.

### Когда использовать

- если нужен весь набор сразу;
- если дальше планируется работать через `NextData()` или `MatrixData(...)`;
- если нужно быстро проверить round-trip после записи.

### Что вернёт

- массив строковых представлений значений;
- для матриц возвращаются маркеры вида `matrix_0`; сами матрицы нужно забирать через `MatrixData(...)`;
- при неверном пароле может вернуть `null`. `[confirmed by code]`

### Ограничения

- метод очищает текущее содержимое объекта и строит его заново по файлу; `[confirmed by code]`
- при чтении файловых вложений библиотека распаковывает zip в кэш-папку;
- поведение при ошибках больше console-oriented, чем exception-oriented.

### Практика

Если вы собираетесь несколько раз итерироваться по данным, лучше один раз вызвать `OpenAll()`, а затем использовать `NextData()`.

<a id="opennextint-i--1"></a>
## `OpenNext(int i = -1)`

```csharp
public string OpenNext(int i = -1)
```

### Что это

Читает одно значение из файла и возвращает его как строку.

### Когда использовать

- когда нужно извлекать данные по одному значению;
- когда вы не хотите сразу работать со всем массивом результатов.

### Как вызвать

```csharp
string first = de.OpenNext();
string fourth = de.OpenNext(3);
```

### Что вернёт

- строковое представление значения;
- для матрицы — маркер `matrix_N`;
- при ошибке или неверном пароле — `null` либо диагностическую строку. `[confirmed by code]`

### Ограничения

- метод на каждом вызове фактически повторно читает файл с начала до нужной позиции; `[confirmed by code]`
- это не "настоящий потоковый reader" с удержанием открытого файла.

### Практика

Используйте `OpenNext()` для редких точечных чтений. Для плотного последовательного перебора он менее выгоден, чем `OpenAll()` + `NextData()`.

<a id="nextdataint-i--1"></a>
## `NextData(int i = -1)`

```csharp
public string NextData(int i = -1)
```

### Что это

Возвращает следующее значение из уже загруженного в память списка ячеек.

### Когда использовать

- после `OpenAll()`;
- после ручного формирования набора через `CreateCell(...)`;
- когда нужен in-memory перебор без повторного чтения файла.

### Что вернёт

Строковое представление следующего значения или маркер `matrix_N` для матрицы.

### Ограничения и подводные камни

- метод не читает файл сам по себе; если список пустой, возвращает строку ошибки `[GDEError - 0002] No data available`; `[confirmed by code]`
- в текущей реализации есть граничный дефект: индексная логика проверяет `DS.NEl >= YList.Count` раньше проверки пустого списка. `[confirmed by code]`

### Практика

Вызывайте `OpenAll()` перед `NextData()`, если данные должны прийти с диска. Для пустого контейнера не рассчитывайте на исключение; ориентируйтесь на возвращаемую строку ошибки.

<a id="dellint-i"></a>
## `Dell(int i)`

```csharp
public void Dell(int i)
```

### Что это

Удаляет одну ячейку из текущего набора в памяти.

### Когда использовать

Когда нужно удалить элемент до очередного `Save()`.

### Что вернёт

Ничего.

### Ограничения и подводные камни

- метод меняет только память, а не уже сохранённые файлы;
- в текущем коде после удаления индексы остальных ячеек пересчитываются некорректно: всем элементам присваивается одно и то же значение. `[confirmed by code]`

### Практика

До исправления реализации безопаснее использовать `ResetData(i)`, если вам нужно удалить элемент из загруженного набора.

<a id="resetdataint-i--1"></a>
## `ResetData(int i = -1)`

```csharp
public void ResetData(int i = -1)
```

### Что это

Очищает весь текущий набор или удаляет одну ячейку по индексу.

### Когда использовать

- перед полной перезаписью контейнера;
- при ручной корректировке уже загруженных данных.

### Что вернёт

Ничего.

### Ограничения

- не изменяет файлы на диске до вызова `Save()`;
- при удалении одного элемента корректно пересчитывает индексы оставшихся ячеек. `[confirmed by code]`

### Практика

Для очистки всего контейнера вызывайте `ResetData()` без параметров. Для удаления одного элемента этот метод сейчас безопаснее, чем `Dell(...)`.

## `Association(bool _TOne)`

```csharp
public void Association(bool _TOne)
```

### Что это

Переключает флаг однофайлового/двухфайлового режима для последующих вызовов публичных методов.

### Когда использовать

Когда один и тот же экземпляр должен начать сохранять и читать в другом режиме.

### Ограничения

- метод не выполняет миграцию уже существующих файлов сам по себе;
- фактический переход происходит только после следующего `Save()`. `[confirmed by code]`

### Практика

Если вы меняете режим хранения, лучше сделать это на новом экземпляре `DEObject` с явной повторной записью данных.

<a id="clonedatadedeobject-de"></a>
## `CloneDataDE(DEObject DE)`

```csharp
public void CloneDataDE(DEObject DE)
```

### Что это

Метод подтягивает данные из другого `DEObject`.

### Как именно работает

- вызывает `DE.OpenAll()`;
- затем присваивает `this.YList = DE.YList`. `[confirmed by code]`

### Почему это важно

Это не глубокое копирование. Оба объекта начинают ссылаться на один и тот же внутренний список. `[confirmed by code]`

### Практика

Если нужна изоляция состояний, не используйте этот метод как полноценный clone. Безопаснее заново создать целевой объект и повторно наполнить его нужными значениями.

<a id="recreatecellint-i-string-types-dynamic-data"></a>
## `RecreateCell(int i, string types, dynamic data)`

```csharp
public void RecreateCell(int i, string types, dynamic data)
```

### Что это

Пересоздаёт ячейку по индексу.

### Когда использовать

Когда нужно заменить тип и/или значение существующей позиции без удаления всего списка.

### Поддерживаемые типы

- `int`
- `double`
- `string`
- `bool`
- `file`

Матрица через этот метод не поддерживается. `[confirmed by code]`

### Ограничения и подводные камни

- для неподдерживаемого строкового типа метод не завершится молча, а оставит тип по умолчанию `integer`; `[confirmed by code]`
- метод работает только если индекс находится в диапазоне списка.

### Практика

Проверяйте входной `types` до вызова. Для матриц проще удалить элемент и создать новый `CreateCell(int[,])`.

## FAQ по `DEObject`

**Нужно ли создавать новый `DEObject` для чтения после `Save()`?**

Не обязательно, но это лучший способ проверить, что данные реально сохранились и читаются как отдельным потребителем.

**Почему `OpenAll()` возвращает строки, даже если исходно это были числа?**

Так устроен публичный контракт библиотеки: наружу возвращается текстовое представление значений, а типизация сохраняется во внутреннем формате и внутреннем списке. `[confirmed by code]`

**Как получить матрицу после чтения?**

Сначала получить строковый маркер `matrix_N`, потом передать его в `MatrixData(...)`.

**Чем `OpenNext()` отличается от `NextData()`?**

`OpenNext()` читает с диска, `NextData()` читает из уже загруженной памяти.

**Можно ли считать `CloneDataDE` полноценным клонированием?**

Нет. По текущему коду это разделение одной и той же внутренней коллекции между двумя объектами. `[confirmed by code]`
