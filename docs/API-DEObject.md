# Класс `DEObject`

`DEObject` — центральный рабочий объект библиотеки GDELib 1.4.0. Через него выполняются все основные действия: создание контейнера, добавление данных, сохранение, повторное открытие и чтение.

## Роль `DEObject`

Если смотреть на библиотеку глазами разработчика, `DEObject` — это единая точка входа. Не нужно отдельно создавать writer, reader или менеджер формата: достаточно одного объекта, который управляет жизненным циклом данных.

## Конструктор

```csharp
DEObject(string path_, bool _TOne = false, string _NameData = "data.sve", string _NameStruct = "struct.sve", string _pathcash = "pathcash")
```

### Для чего нужен

Конструктор задаёт:

- рабочую папку контейнера;
- режим хранения;
- имена файлов;
- путь к кэш-папке.

### Параметры

| Параметр | Назначение |
| --- | --- |
| `path_` | Рабочая папка, где находятся или будут созданы файлы SVE |
| `_TOne` | Режим хранения: `false` — двухфайловый, `true` — однофайловый |
| `_NameData` | Имя файла данных или общего контейнера |
| `_NameStruct` | Имя файла структуры в двухфайловом режиме |
| `_pathcash` | Путь к папке кэша для работы с файлами |

### Пример

```csharp
var de = new DEObject(storeDir);
var oneFile = new DEObject(storeDir, true, "bundle.sve", "struct.sve", storeDir);
```

## `CreateCell(string, object)`

Добавляет новую ячейку в контейнер.

### Поддерживаемые типы

- `int`
- `double`
- `string`
- `bool`
- `file`

### Примеры

```csharp
de.CreateCell("int", 10);
de.CreateCell("double", 15.75);
de.CreateCell("string", "Документ");
de.CreateCell("bool", true);
de.CreateCell("file", pathToFile);
```

### Когда использовать

Метод вызывается каждый раз, когда нужно добавить ещё одно значение в конец контейнера.

### Что важно знать

- порядок вызовов `CreateCell(...)` определяет порядок чтения;
- в файловом сценарии в метод передаётся путь к исходному файлу, но в контейнер сохраняется сам файл.

## `Save()`

Сохраняет текущий набор данных на диск.

### Когда использовать

После того как контейнер заполнен нужными ячейками.

### Пример

```csharp
de.Save();
```

### Практика

После `Save()` можно сразу вызвать `OpenAll()` на том же объекте либо создать новый экземпляр `DEObject` и прочитать данные из той же папки.

## `OpenAll()`

Читает весь набор данных из контейнера.

### Что возвращает

Массив строк, содержащий значения по порядку хранения.

### Пример

```csharp
string[] values = de.OpenAll();
```

### Типичный сценарий

```csharp
var de = new DEObject(storeDir);
de.CreateCell("int", 42);
de.CreateCell("string", "Пример");
de.Save();

string[] values = de.OpenAll();
```

### Особенность для файлов

Если в контейнере был сохранён файл, `OpenAll()` вернёт путь к восстановленной копии файла в папке кэша.

## `OpenNext(int)`

Читает одно значение из контейнера.

### Когда использовать

Когда не нужно получать весь набор сразу, а достаточно отдельных значений.

### Примеры

```csharp
string first = de.OpenNext(0);
string second = de.OpenNext(1);
```

### Что возвращает

Строковое представление значения по выбранной позиции.

## `ResetData(int)`

Управляет текущим набором данных в памяти.

### Варианты использования

```csharp
de.ResetData();     // Очистить весь набор
de.ResetData(2);    // Удалить один элемент
```

### Когда полезно

- перед повторным заполнением контейнера;
- при подготовке нового варианта данных перед следующим `Save()`.

## `Dell(int)`

Удаляет элемент по индексу.

### Пример

```csharp
de.Dell(0);
```

### Когда использовать

Когда нужно убрать одну конкретную ячейку из текущего набора.

## `RecreateCell(int, string, object)`

Пересоздаёт ячейку по индексу.

### Пример

```csharp
de.RecreateCell(1, "string", "Новое значение");
```

### Когда использовать

Если нужно не просто удалить и добавить элемент заново, а заменить его на месте.

## `NumberInfo()`

Возвращает количество ячеек в контейнере.

### Пример

```csharp
string count = de.NumberInfo();
Console.WriteLine(count);
```

### Когда использовать

Когда нужно показать число записей в интерфейсе или вывести его в лог.

## `Association(bool)`

Переключает режим хранения.

### Пример

```csharp
de.Association(true);
```

### Практический смысл

Этот метод удобен, если в приложении нужно переключиться между двухфайловым и однофайловым вариантом работы.

## `CloneDataDE(DEObject)`

Копирует данные из другого объекта `DEObject`.

### Пример

```csharp
var source = new DEObject(sourceDir);
var target = new DEObject(targetDir);

target.CloneDataDE(source);
```

### Когда использовать

Когда данные уже собраны в другом объекте и их нужно перенести в текущий рабочий экземпляр.

## Типовые сценарии работы

### Запись и чтение тем же объектом

```csharp
var de = new DEObject(storeDir);
de.CreateCell("int", 7);
de.CreateCell("string", "same object");
de.Save();

string[] values = de.OpenAll();
```

### Запись и чтение новым объектом

```csharp
var writer = new DEObject(storeDir);
writer.CreateCell("int", 7);
writer.Save();

var reader = new DEObject(storeDir);
string[] values = reader.OpenAll();
```

### Работа с файлом

```csharp
var de = new DEObject(storeDir);
de.CreateCell("file", pathToFile);
de.Save();

string restoredFilePath = de.OpenAll()[0];
```

## Лучшие практики

- Используйте понятные имена типов в `CreateCell(...)`: `int`, `double`, `string`, `bool`, `file`.
- Сразу определяйтесь, нужен ли проекту однофайловый или двухфайловый режим.
- Для файловых ячеек работайте с путём, который вернулся после чтения, а не со старым исходным путём.
- Если нужно быстро проверить сохранение, достаточно вызвать `OpenAll()` сразу после `Save()`.

## Короткий FAQ по `DEObject`

**Нужно ли каждый раз создавать новый объект для чтения?**

Нет. Можно читать тем же экземпляром, которым данные были сохранены.

**Можно ли хранить файл внутри контейнера?**

Да. Для этого используется `CreateCell("file", pathToFile)`.

**Что возвращает `OpenAll()`?**

Массив строковых значений в порядке их сохранения.

**Когда использовать `OpenNext(...)` вместо `OpenAll()`?**

Когда нужно получить одно конкретное значение, а не весь набор сразу.
