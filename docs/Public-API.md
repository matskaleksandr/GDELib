# Карта публичного API

В версии `1.4.0` работа с GDELib строится вокруг одного основного публичного класса — `DEObject`.

## Публичные типы

| Тип | Назначение |
| --- | --- |
| `DEObject` | Создание контейнера, добавление ячеек, сохранение и чтение данных |

## Иерархия

```text
System.Object
└── GDELib.DEObject
```

## Основные члены `DEObject`

| Член | Назначение |
| --- | --- |
| `DEObject(string, bool, string, string, string)` | Создаёт объект для работы с контейнером |
| `CreateCell(string, object)` | Добавляет новую ячейку данных |
| `Save()` | Сохраняет текущий набор данных |
| `OpenAll()` | Читает весь набор данных |
| `OpenNext(int)` | Читает одно значение по позиции |
| `ResetData(int)` | Очищает весь набор или удаляет один элемент |
| `Dell(int)` | Удаляет элемент по индексу |
| `RecreateCell(int, string, object)` | Пересоздаёт ячейку по индексу |
| `NumberInfo()` | Возвращает количество ячеек |
| `Association(bool)` | Переключает режим хранения |
| `CloneDataDE(DEObject)` | Копирует данные из другого объекта |

## Что можно передавать в `CreateCell`

В `1.4.0` метод `CreateCell(...)` используется для следующих пользовательских типов:

- `int`
- `double`
- `string`
- `bool`
- `file`

Тип задаётся строкой, например:

```csharp
de.CreateCell("int", 42);
de.CreateCell("string", "text");
de.CreateCell("file", pathToFile);
```

## Базовые методы `System.Object`

Как и любой обычный класс .NET, `DEObject` также наследует стандартные методы базового класса:

- `ToString()`
- `GetType()`
- `Equals(object)`
- `GetHashCode()`

Эти методы доступны и могут вызываться обычным образом, но прикладной API библиотеки строится именно вокруг методов `DEObject`.

## Практическая карта использования

Обычно код с GDELib выглядит так:

1. создать `DEObject`;
2. добавить ячейки через `CreateCell(...)`;
3. вызвать `Save()`;
4. прочитать данные через `OpenAll()` или `OpenNext(...)`.

Подробное описание каждого метода находится в [API-DEObject.md](API-DEObject.md).
