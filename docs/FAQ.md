# FAQ

## Что делает GDELib?

GDELib 1.4.0 сохраняет и читает данные приложения в формате SVE через класс `DEObject`.

## Когда библиотеку удобно использовать?

Когда нужен простой контейнер для локального хранения данных: настроек, сохранений, служебных записей и файлов.

## Чем отличается подключение через NuGet от прямого подключения DLL?

### NuGet

- удобнее обновлять версию;
- проще подключать библиотеку в новый проект;
- проще повторять установку в команде и CI.

### DLL

- подходит для офлайн-поставок;
- удобна, когда пакет подключается вручную;
- требует выбрать правильную сборку под нужную платформу.

## С чего начать работу?

Минимальный путь такой:

1. установить `GDELib 1.4.0`;
2. создать рабочую папку;
3. создать `DEObject`;
4. добавить данные через `CreateCell(...)`;
5. вызвать `Save()`;
6. прочитать данные через `OpenAll()` или `OpenNext(...)`.

## Нужно ли создавать новый `DEObject` для чтения?

Нет. Можно использовать тот же экземпляр, которым данные были сохранены.

## Как устроен `DEObject`?

Это основной рабочий объект библиотеки. Через него выполняются запись, чтение и управление набором ячеек.

## Какие типы данных поддерживаются в версии 1.4.0?

- `int`
- `double`
- `string`
- `bool`
- `file`

## Что означает `CreateCell("file", path)`?

В метод передаётся путь к исходному файлу. При этом библиотека сохраняет сам файл внутри контейнера.

## Что я получу при чтении файловой ячейки?

При чтении библиотека возвращает путь к восстановленной копии файла в папке кэша.

## Как понять, что данные сохранились корректно?

После `Save()` вызовите `OpenAll()` и проверьте, что значения читаются в ожидаемом порядке.

## Чем отличаются однофайловый и двухфайловый режимы?

### Двухфайловый режим

- хранит структуру и данные отдельно;
- по умолчанию использует `struct.sve` и `data.sve`.

### Однофайловый режим

- хранит всё в одном основном файле;
- удобен, когда проекту нужен один контейнер.

## Какой режим лучше выбрать?

Если нужен максимально простой и наглядный старт, чаще выбирают двухфайловый режим. Если проекту нужен один основной контейнер, выбирают однофайловый.

## Что возвращают `OpenAll()` и `OpenNext(...)`?

Они возвращают строковые значения в том виде, в котором библиотека представляет их при чтении.

## Где лежат восстановленные файлы?

Они размещаются в кэш-папке, связанной с контейнером.

## Можно ли хранить в одном контейнере и данные, и файлы?

Да. Это один из типовых сценариев использования GDELib 1.4.0.

## Какие методы относятся к рабочему API библиотеки?

Основные методы — это:

- `CreateCell(...)`
- `Save()`
- `OpenAll()`
- `OpenNext(...)`
- `ResetData(...)`
- `Dell(...)`
- `RecreateCell(...)`
- `NumberInfo()`
- `Association(...)`
- `CloneDataDE(...)`

## Что делать, если нужно быстро начать без долгого изучения формата?

Используйте следующий шаблон:

1. создайте `DEObject`;
2. добавьте ячейки через `CreateCell(...)`;
3. сохраните данные;
4. сразу прочитайте их обратно через `OpenAll()`.
