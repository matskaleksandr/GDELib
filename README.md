# GDELib 1.4.0

GDELib 1.4.0 — это библиотека для .NET, которая помогает сохранять и читать данные в формате SVE через простой прикладной API. Работа с библиотекой строится вокруг одного класса — `DEObject`.

Библиотека подходит для сценариев, где нужно быстро организовать локальное хранение данных приложения: настроек, игровых сохранений, служебных записей и встроенных файловых ресурсов.

Проект распространяется как open-source библиотека под лицензией MIT.

## Что умеет библиотека

- сохранять `int`, `double`, `string`, `bool` и файлы;
- работать в двух режимах:
  - двухфайловый режим — структура и данные хранятся отдельно;
  - однофайловый режим — структура и данные сохраняются в одном файле;
- читать весь набор данных сразу или получать значения по одному;
- использовать один и тот же экземпляр `DEObject` и для записи, и для последующего чтения;
- сохранять файл целиком внутри контейнера, а не только путь к нему.

## Когда использовать GDELib

GDELib удобно применять, когда нужен простой файловый контейнер с понятным вызовом методов из кода:

- в настольных приложениях;
- в игровых проектах;
- в утилитах и небольших сервисах;
- в проектах, где нужно хранить данные и связанные с ними файлы в одном прикладном формате.

## Поддерживаемые платформы

Пакет `GDELib 1.4.0` опубликован для:

- `.NET 6.0`
- `.NET Standard 2.1`
- `.NET Framework 4.7.2`

## Установка через NuGet

Package Manager Console:

```powershell
Install-Package GDELib -Version 1.4.0
```

.NET CLI:

```bash
dotnet add package GDELib --version 1.4.0
```

`PackageReference`:

```xml
<ItemGroup>
  <PackageReference Include="GDELib" Version="1.4.0" />
</ItemGroup>
```

## Подключение через `.dll`

Если библиотеку удобнее подключать вручную, возьмите `GDELib.dll` из одной из папок пакета:

- `lib/net6.0`
- `lib/netstandard2.1`
- `lib/net472`

После этого добавьте ссылку на DLL в свой проект и выберите сборку, соответствующую целевой платформе приложения.

## Минимальный пример

```csharp
using System;
using System.IO;
using GDELib;

string storeDir = Path.Combine(Environment.CurrentDirectory, "Store");
Directory.CreateDirectory(storeDir);

var de = new DEObject(storeDir);
de.CreateCell("int", 42);
de.CreateCell("string", "Привет, GDELib");
de.CreateCell("bool", true);
de.Save();

string[] values = de.OpenAll();
Console.WriteLine(string.Join(", ", values));
```

В этом сценарии новый экземпляр `DEObject` не обязателен: библиотека позволяет сохранить данные и затем сразу прочитать их тем же объектом.

## Пример с файлом

Ниже показан важный сценарий версии `1.4.0`: в контейнер сохраняется сам файл, а при чтении библиотека возвращает путь к восстановленной копии в папке кэша.

```csharp
using System;
using System.IO;
using GDELib;

string storeDir = Path.Combine(Environment.CurrentDirectory, "Store");
Directory.CreateDirectory(storeDir);

string sourceFile = Path.Combine(storeDir, "note.txt");
File.WriteAllText(sourceFile, "Пример содержимого файла");

var de = new DEObject(storeDir);
de.CreateCell("file", sourceFile);
de.Save();

string[] values = de.OpenAll();
string restoredFilePath = values[0];

Console.WriteLine(restoredFilePath);
```

## Что создаётся на диске

По умолчанию библиотека использует:

- `struct.sve` — файл структуры в двухфайловом режиме;
- `data.sve` — файл данных;
- `cashfile` — папку кэша для работы с файлами.

Если включить однофайловый режим, основным файлом будет один контейнер данных, например `data.sve` или любое имя, переданное в конструктор.

## Короткий FAQ

**Можно ли читать тем же объектом, которым данные были сохранены?**

Да. Для типового сценария достаточно вызвать `Save()`, а затем `OpenAll()` или `OpenNext(...)` на том же экземпляре `DEObject`.

**Что передавать в `CreateCell("file", ...)`?**

Нужно передать путь к исходному файлу. Внутри контейнера сохраняется сам файл, а при чтении библиотека возвращает путь к его восстановленной копии.

**Когда выбирать однофайловый режим?**

Когда нужен один основной файл контейнера. Если удобнее держать структуру и данные раздельно, используйте двухфайловый режим.

**Есть ли у библиотеки много публичных классов?**

Нет. Основной внешний API версии `1.4.0` построен вокруг класса `DEObject`.

## Что важно помнить при использовании

- Сразу задавайте понятную рабочую папку для контейнера.
- Если сохраняете файлы, храните исходный путь только как источник данных; при чтении используйте путь, который вернула библиотека.
- Если проекту нужен один файл контейнера, включайте однофайловый режим через параметр `_TOne`.
