# Examples

В этой папке лежит небольшой runnable-набор для сравнения опубликованных релизов GDELib:

- `GDELib.Examples.V0_1_4` — пример на DLL пакета `0.1.4`;
- `GDELib.Examples.V1_4_0` — пример на DLL пакета `1.4.0`.

Оба проекта используют один и тот же сценарий:

1. выводят публичный API `DEObject`;
2. выполняют сценарий двухфайлового режима;
3. выполняют сценарий однофайлового режима.

## Что это даёт

Такой набор полезен в двух задачах:

- быстро проверить реальное поведение опубликованной DLL;
- сравнить релизы `0.1.4` и `1.4.0` не по markdown, а по рабочему коду.

## Как запустить

Нужен `.NET 6 SDK`.

Из корня репозитория:

```bash
dotnet run --project examples/GDELib.Examples.V0_1_4/GDELib.Examples.V0_1_4.csproj
dotnet run --project examples/GDELib.Examples.V1_4_0/GDELib.Examples.V1_4_0.csproj
```

## Что создаётся при запуске

Во время выполнения примеры создают служебные папки в `examples/runtime/`:

- `examples/runtime/0.1.4`
- `examples/runtime/1.4.0`

Внутри можно посмотреть:

- результат двухфайлового сценария;
- результат однофайлового сценария;
- служебные артефакты, связанные с файловыми ячейками.

## Что смотреть при сравнении

- одинаков ли внешний API `DEObject`;
- какие файлы создаются в двухфайловом режиме;
- как выглядит однофайловый режим;
- как библиотека возвращает значения после `OpenAll()` и `OpenNext(...)`.
